

Durante el desarrollo del proyecto Monitor de Red, el repositorio en GitHub fue utilizado no solo para almacenar los archivos del sistema, sino también como una herramienta de seguimiento del avance del proyecto, permitiendo relacionar cada cambio con una actividad específica de la metodología utilizada.



En la fase de Cascada se realizaron los primeros commits, los cuales estuvieron enfocados en la planificación, el análisis y el diseño del sistema. En esta etapa se crearon archivos relacionados con la estructura general del proyecto, la definición de requerimientos y el modelo de base de datos en MySQL. Estos commits sentaron las bases para el desarrollo posterior del sistema.



Posteriormente, en la etapa de Scrum, se realizaron diferentes commits de manera incremental, representando pequeños avances del proyecto en varios sprints. Entre estos cambios se incluyen archivos relacionados con la documentación de la metodología, la implementación del monitoreo de red, la organización del proyecto y la creación del archivo modelo\_hibrido.md.



Un ejemplo importante es el commit llamado “Archivo modelo\_hibrido”, el cual corresponde a la actividad de definición del modelo híbrido utilizado en el proyecto. Otros commits reflejan avances en la fase de implementación y documentación, mostrando cómo el proyecto fue creciendo de forma gradual.



Gracias al uso de Git, fue posible llevar un control claro y ordenado de la evolución del proyecto, permitiendo visualizar el progreso desde la planeación hasta la documentación final, demostrando que Git puede ser utilizado también como una herramienta de seguimiento del proyecto.
